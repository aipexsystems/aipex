# ADR-001: AIPEX Repository Architecture

Status: Accepted

Date: 2026-07-01

## Context

AIPEX intends to evolve from services into products and eventually ecosystems. The repository must support knowledge accumulation, reusable intellectual property, and operational scaling.

## Decision

Adopt a structure separating:

- services
- products
- clients
- templates
- playbooks
- knowledge
- case studies

## Consequences

### Positive

- Faster onboarding
- Reusable assets
- Strong documentation culture
- Easier productization

### Negative

- Slightly higher initial complexity
- Requires enforcement discipline
